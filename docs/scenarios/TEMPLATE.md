## 작성 가이드

### 성능 지표 (필수)

| 지표명 | 단위 | 설명 |
| --- | --- | --- |
| p95 응답시간 | ms | 95% 요청의 응답시간 상한 |
| 평균 응답시간 | ms | 전체 요청의 평균 |
| 처리량 | req/s | 초당 처리 요청 수 |
| 에러율 | % | 실패 요청 비율 |

### 리소스 지표 (선택)

| 지표명 | 단위 | 설명 |
| --- | --- | --- |
| CPU | % | 컨테이너 CPU 사용률 |
| RAM | MB | 메모리 사용량 (필요시에만) |

### 작성 원칙

1. **상황**: 2-3줄로 문제 맥락만 설명
2. **문제점/분석**: 불릿으로 핵심만 (3-5개)
3. **개선 비율**: 배수(N배) 또는 퍼센트(N%) 중 하나로 통일
4. **숫자 정밀도**: 소수점 없이 정수로 반올림 (예: 867ms, 46req/s)
5. **단위 표기**: 값 뒤에 단위 붙임 (예: `867ms`, `46req/s`, `86%`)
6. **개선 단계 비교**: 반드시 직전 단계와 동일한 조건으로 비교
   - 개선 1: 기존 시스템과 비교
   - 개선 2: 개선 1의 최종 설정 기준으로 비교
   - 개선 N: 개선 N-1의 최종 설정 기준으로 비교
   - 예: 개선 2에서 2워커로 테스트했다면, 개선 3도 2워커 기준으로 측정
